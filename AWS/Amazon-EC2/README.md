# EC2
- EC2란 Amazon Elastic Compute Cloud의 줄임말로서 AWS에서 제공하는 클라우드 컴퓨팅. 독립된 컴퓨터를 임대해주는 서비스
- 또한 EC2는 복잡한 공유기 세팅없이 인터넷을 통해서 자유롭게 접속할 수 있고, 이미지(AMI) 기능도 사용할 수 있음.
- AMI : 컴퓨터를 사용시 프로그램 설치, 파일 저장, 설정 변경등을 하게 되는데, 이 OS 상태 그대로 저장하는 기능.
- EC2를 통해 원하는 만큼 가상 서버를 구축하고 보안 및 네트워크 구성과 스토리지 관리가 가능하며, 요구 사항이나 갑작스러운 트래픽 증가 등 변동 사항에 따라 신속하게 규모를 확장, 축소해 서버 트래픽 예측 필요성이 줄어든다는 장점이 있음.

#### 특징
- 컴퓨팅 요구사항의 변화에 따라 컴퓨팅 파워를 조정 가능.
- 실제로 사용한 용량 만큼만 지불.
- Linux / Windows 중 OS 선택이 쉽게 가능하다
- 몇 분이면 전세계에 컴퓨터 수백여대를 생성 가능.
- 머신러닝, 웹서버, 게임서버, 이미지 처리 등 다양한 용도에 최적화된 서버를 쉽게 구성 가능
- 여러 다른 AWS 서비스와의 유기적인 연동이 가능


### EC2의 구성 (Instance / EBS / AMI)
- 컴퓨팅에 해당하는 인스턴스
- 하드디스크에 해당하는 EBS
- 랜카드에 해당하는 ENI

### EC2 인스턴스
- 인스턴스란, aws 클라우드에서 사용하는 가상 컴퓨터
- 가상의 컴퓨터에의 CPU, 메모리, 그래픽카드 등 연산을 위한 하드웨어 부분을 담당
- 애플리케이션 종류와 목적에 따라 적합한 리소스 조합을 선택할 수 있는 유연성을 제공하는 서비스
- 고사양 게임용, 서버용, 대량의 데이터 저장소, 영상용, 로봇 등 컴퓨터 하드웨어 구성에 따라 여러 사용 목적을 정할 수 있다.


### EBS(Elastic Block Storage)
- 인스턴스가 연산에 관한 (CPU,메모리 등) 처리를 한다고 하면, 데이터를 저장하는 역할 EBS
- 클라우드에서 사용하는 가상 하드디스크(HDD).
- EBS는 AWS 클라우드의 Amazone EC2 인스턴스에 사용할 영구 블록 스토리지 볼륨을 제공.
- 그리고 단 몇분 내에 사용량을 많게 또는 적게 확장할 수 있으며, 프로비저닝(빌리는 행위)한 부분에 대해서만 저렴한 비용을 지불 할 수 있다.


### AMI(Amazon Machine Image)
- AMI는 EC2 인스턴스를 실행하기 위한 정보를 모은 단위 
- EC2(가상 컴퓨터)를 실행하기 위해서는 CPU 프로세서 타입, 저장공간 용량, 32비트, 64비트, 윈도우, 리눅스, 소프트웨어 설치..등 정보가 필요한데 이러한 세팅 정보(템플릿)를 저장한 단위.
